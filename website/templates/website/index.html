Documentation de l'API

Voici la documentation complète de l'API que nous avons développée.

1. Authentification

Cette section couvre les routes pour l'inscription et la connexion.

POST /register/

Permet de créer un nouvel utilisateur.

Paramètres requis :
  • username : Nom d'utilisateur (string).
  • password : Mot de passe (string).
  • email : Adresse email (string).
Exemple de Réponse (Succès) :
{
    "message": "Utilisateur créé avec succès."
}
POST /login/

Permet de se connecter avec un compte existant.

Paramètres requis :
  • username : Nom d'utilisateur (string).
  • password : Mot de passe (string).
  • email : Adresse email (string).
Exemple de Réponse (Succès) :
{
    "refresh": "eyJhbGciOiJIUzI1NiIs...",
    "access": "eyJhbGciOiJIUzI1NiIs..."
}

2. Questions

Cette section couvre les routes pour gérer les questions.

GET /questions/

Récupère la liste des questions existantes.

Exemple de Réponse :
[
    {
        "id": 1,
        "title": "Comment fonctionne l'API ?",
        "body": "Explication sur l'API...",
        "author": "chat7",
        "upvotes": 0,
        "downvotes": 0,
        "created_at": "2025-01-08T00:00:00Z"
    }
]
POST /questions/

Permet de créer une nouvelle question.

Paramètres requis :
  • title : Titre de la question (string).
  • body : Contenu de la question (string).
Exemple de Réponse (Succès) :
{
    "id": 1,
    "title": "Comment fonctionne l'API ?",
    "body": "Explication sur l'API...",
    "author": "chat7",
    "upvotes": 0,
    "downvotes": 0,
    "created_at": "2025-01-08T00:00:00Z"
}
DELETE /questions/{id}/

Permet de supprimer une question (seulement par son auteur).

Exemple de Réponse (Succès) :
{
    "message": "Suppression de la question effectuée avec succès."
}
POST /questions/{id}/upvote/

Permet de voter positivement pour une question.

Exemple de Réponse (Succès) :
{
    "message": "Question upvotée",
    "upvotes": 5
}
POST /questions/{id}/downvote/

Permet de voter négativement pour une question.

Exemple de Réponse (Succès) :
{
    "message": "Question downvotée",
    "downvotes": 3
}

3. Réponses

Cette section couvre les routes pour gérer les réponses.

GET /answers/

Récupère la liste des réponses existantes.

Exemple de Réponse :
[
    {
        "id": 1,
        "question": 1,
        "body": "Voici une réponse à la question.",
        "author": "chat7",
        "upvotes": 0,
        "downvotes": 0,
        "created_at": "2025-01-08T00:00:00Z"
    }
]
POST /answers/

Permet de créer une nouvelle réponse à une question.

Paramètres requis :
  • question : ID de la question (integer).
  • body : Contenu de la réponse (string).
Exemple de Réponse (Succès) :
{
    "id": 1,
    "question": 1,
    "body": "Voici une réponse à la question.",
    "author": "chat7",
    "upvotes": 0,
    "downvotes": 0,
    "created_at": "2025-01-08T00:00:00Z"
}
DELETE /answers/{id}/

Permet de supprimer une réponse (seulement par son auteur).

Exemple de Réponse (Succès) :
{
    "message": "Suppression de la réponse effectuée avec succès."
}
POST /answers/{id}/upvote/

Permet de voter positivement pour une réponse.

Exemple de Réponse (Succès) :
{
    "message": "Réponse upvotée",
    "upvotes": 7
}
POST /answers/{id}/downvote/

Permet de voter négativement pour une réponse.

Exemple de Réponse (Succès) :
{
    "message": "Réponse downvotée",
    "downvotes": 4
}

4. Édition de Profil

Cette section couvre la route pour modifier les informations d'un utilisateur connecté.

PUT /profile/edit/

Permet de modifier les informations de l'utilisateur connecté.

Paramètres requis :
  • username : Nouveau nom d'utilisateur (string, optionnel).
  • email : Nouvelle adresse email (string, optionnel).
  • password : Nouveau mot de passe (string, optionnel).
Exemple de Réponse (Succès) :
{
    "message": "Profil mis à jour avec succès."
}
